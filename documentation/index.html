Codeigniter Starter Kit is a development kit primarily for developers but anybody can build a robust web application out of it.

Quick start guide

After downloading the application, copy the steller-color or steller-white folder in your local or remote server placing in the htdocs folder or the root folder your server serving the site from. Create a database with the name of your choice. Then go to Application > config > database.php and fill in the database details, i.e., the database username, password, host address (mostly localhost) and the database name. Then go to the browser and Sign in with the given username and passworrd. Initial username/email is [admin@admin.com] and password is [abc123] for ADMINs.

 

 

Steps

  • Add user from the Add user submenu of the Users menu located in the left sidebar.
  • Set a role for the user, either user or admin level previledge.
  • Add color, size, category and subcategory from the Products menu to be able to add producs.
  • Add products from the Add product submenu.
  • All the users and products are listed in the respective menu table. Click on them to go to individual profile or product page.
  • Add site logo and customize the settings of the site to your taste from the customization table.
  • You can backup the database from the Backup menu.
  • Use the HTML static pages to build new dynamic pages.
  • The rest of the documentation details each f these features.

Installation

  • Server requirements
    • Apache/nginx with PHP and MySQL server
    • PHP v5.3+
  • Installation guide
    • Unzip the Codeigniter-starter-kit-DotDev.zip you have downloaded from Envato market.
    • Upload the files to your server using an FTP server
    • The structure should look like this:
      • /application
      • /documentation
      • /files
      • /system
      • /index.php
    • Create a MySQL Database via your cPanel/server command line if you have access to. Remember these while creating the Database:
      • Database name
      • Database username
      • Database password
    • Go to application/database and update the info.
    • Open the browser and enetr the URL of your domain/localhost: http://www.domain.com or http://www.domain.com/foldername
    • You should get the login page
    • That’s it!

Initial setup

 

  • Go to Configuration, add the logo, set the site title and description
  • Add copyright info for footer and set the product currency.
  • Click on the top right profile image dropdown and change your password.

Login/Registration

 

  • You can login with the super user credentials or create a new account. The admins can set the type as normal user or admin. The new registered users will have to verify the email by clicking on the emailed link that the email exists.
  • You can reset your password if you forgot it clicking the forgot password link or you can change the password from your profile once you are logged in too.

Dashboard

 

  • In the dashboard you can add notes or reminders with the quick notes widgets. You can set them as done when completed. All on the fly! Without even reloading the page and the data will be persisted to database, not in the browser localstorage.
  • The top widgents are dynamically updated. Look into the respective code for details.
  • The charts and map, you can change the parameters from the source code. Nothing fancy there.

Users management

 

     1.List user menu lists all the users. You can view individual user’s basic info from the listed table, go to individual profile or delete a particular user. You can export the list as CSV/PDF or print it from right there.

 

 

     2. From the add user menu, you can add a new user with username and email along with some other basic info and a profile picture. The profile picture is previewed as soon as you upload one.

 

 

     3. In the user profile page, you can edit user info as admin, the user or any other user can leave a comment with image attachment. All the individual comments will be listed in a descending order. The uploaded images can be viewed in a lightbox gallery. The user can edit the info from his profile page too.

Role management

 

From the Users role menu, the admin previledged users can change the role of an user from admin to user and vice versa. You can see both the Admins and Users list here. The users cannot access this page, only the admins can.

Products management

 

  • List products menu under the Producs main menu lists all the products added. You can view individual product’s detailed info from the listed table, go to individual product page or delete a particular product. You can export the list as CSV/PDF or print it from right there. You can search the table live as a part of Datatable implementation.
    • To go to the products listing table.
    • Click on the names or the image to visit the detailed product info, the individual page.
    • Search table live.


       
  • If you want to create a new product, you have to first add some categories and subcategories, add color, sized and brands to be used while setting these input fields to be generated i the add product form. All these options can be added from separate submenu under the Product menu. You can add new fields, delete them or edit them, and even set them as active or inactive from each respective submenu page. Static HTML uses

 

All the pages used in the dynamic protion of the site has an equivalent HTML version so that these pages can be implemented very easily. The HTML portion has some extra non-dynamic implementation too, for example, dropzone, form wizards etc.

Application structure

  • Application - the main Codeigniter folder to place MVC files.
  • Config - Config files to config databases, autoloads etc.
  • Controllers - The controllers with all the functions which constitutes the app
  • Models - Database model files
  • Views - view files which show up in the admin panel
  • Backend - All the view files of the dynamic admin panel.
  • Static-html - All the view files of the static HTML pages.
  • Assets - assets for the app - css, fonts, js, images
  • Static assets - are the assets for the static html pages, seperated for use convenience.
  • Documentation
  • System

For developers

Sample Code for database insert - if the user has logged in and has proper permission, we will load the user-data (his profile thumb, name, email etc. for page top menu) and view him the add user form. Please refer to https://www.codeigniter.com/docs for detailed file structure and documentation.
Below are just sample code snippets. There are two controllers, login for login controls and crud for dashboard, user, product and all other logged in features management. The code structure has been kept very simple and straightforward without any complexity for any programmers to copy and paste from. All the functions has comments.
<?php /*Add user Form View*/
public function Add_User() {
    if ($this->session->userdata('user_login_access') != False) {
        $data['settingsvalue'] = $this->crud_model->GetSettingsValue();
        $this->load->view('backend/adduser', $data);
    } else {
        redirect(base_url(), 'refresh');
    }
} ?>

<?php /*Get the current values to pre-populate the form*/

public function GetSettingsValue() {
    $settings = $this->db->dbprefix('settings');
    $sql      = "SELECT * FROM $settings";
    $query    = $this->db->query($sql);
    $result   = $query->row();
    return $result;
} ?>
